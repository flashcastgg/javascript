1. Foundations & History

What is JavaScript? Beginner

JavaScript (JS) is a lightweight, interpreted, multi-paradigm programming language. It is the only language that runs natively in web browsers, making it the backbone of the web alongside HTML and CSS.

Key traits:

  • Dynamically typed — types are determined at runtime
  • Single-threaded — one call stack, one task at a time
  • Event-driven — responds to user interactions, timers, network events
  • First-class functions — functions are values, can be passed around
  • Prototype-based — inheritance via prototype chains, not classical OOP
  • Runs in browsers AND server (Node.js), mobile (React Native), desktop (Electron)
Brief History Beginner
  • 1995 — Brendan Eich creates JS in 10 days at Netscape. Originally called Mocha, then LiveScript, then JavaScript.
  • 1997 — ECMAScript 1 standardized by ECMA International (TC39 committee).
  • 2009 — ES5: strict mode, JSON, Array methods.
  • 2009 — Node.js released. JS escapes the browser.
  • 2015 — ES6/ES2015: biggest update ever. Classes, arrow functions, let/const, Promises, modules.
  • 2016–present — Yearly releases: ES2016, ES2017, … ES2024.
How JavaScript Runs Beginner

JavaScript runs inside a JavaScript Engine. Every browser has one:

V8

Google Chrome, Node.js, Edge (Chromium)

SpiderMonkey

Mozilla Firefox

JavaScriptCore

Safari, iOS WebKit

Hermes

React Native (Meta)

The engine parses → compiles (JIT) → executes your code. Modern engines use Just-In-Time compilation to turn JS into optimized machine code at runtime.

Runtime environment adds: Web APIs (setTimeout, fetch, DOM), Event Loop, Callback Queue, Microtask Queue.

5. Functions

Function Declaration vs Expression Beginner
// Function Declaration — hoisted, usable before defined function greet(name) { return `Hello, ${name}!`; } // Function Expression — not hoisted const greet = function(name) { return `Hello, ${name}!`; }; // Arrow Function (ES6) — concise, no own 'this' const greet = (name) => `Hello, ${name}!`; const double = n => n * 2; // single param, implicit return const add = (a, b) => a + b; // IIFE — Immediately Invoked Function Expression (function() { console.log("runs immediately"); })();
Parameters & Arguments Beginner
// Default parameters function log(msg, level = "info") { console.log(`[${level}] ${msg}`); } // Rest parameters — collects remaining args into array function sum(...nums) { return nums.reduce((a, b) => a + b, 0); } // arguments object (old way, not in arrow functions) function old() { console.log(arguments); // array-like, not a real array } // Destructuring in params function display({ name, age = 0 }) { console.log(name, age); }
Higher-Order Functions Intermediate

6. Scope & Closures

Scope Types Intermediate
// Global scope — accessible everywhere var globalVar = "I'm global"; // Function scope function myFn() { let local = "only here"; } // Block scope (let/const) { let blockVar = "only in this block"; const also = "block scoped"; } // Lexical scope — inner functions see outer scope function outer() { let x = 10; function inner() { console.log(x); // 10 — can see x from outer } }
Closures Intermediate

A closure is a function that remembers its lexical scope even when called outside that scope. This is one of the most powerful features of JavaScript.

function makeCounter() { let count = 0; return { increment: () => ++count, decrement: () => --count, value: () => count }; } const counter = makeCounter(); counter.increment(); // 1 counter.increment(); // 2 counter.value(); // 2 — count is private, only accessible via methods
✅ Best Practice Use closures to create private state, module patterns, and factory functions. They are essential for understanding React hooks, event listeners, and memoization.
Hoisting Intermediate
// Function declarations are fully hoisted hello(); // ✅ works before declaration function hello() { console.log("hi"); } // var is hoisted but value is not console.log(x); // undefined (not ReferenceError) var x = 5; // let/const are NOT hoisted (Temporal Dead Zone) console.log(y); // ❌ ReferenceError let y = 5;

8. Arrays

Array Basics & Mutation Methods Beginner
const arr = [1, 2, 3]; // Add / remove arr.push(4); // add to end → [1,2,3,4] arr.pop(); // remove from end arr.unshift(0); // add to start → [0,1,2,3] arr.shift(); // remove from start arr.splice(1, 2); // remove 2 items at index 1 arr.splice(1, 0, "a"); // insert "a" at index 1 // Info arr.length; arr.indexOf(2); // index of value, -1 if not found arr.includes(2); // true/false arr.findIndex(x => x > 2); // index of first match
Functional Array Methods Intermediate

These methods return a new array and never mutate the original. They are essential for modern JavaScript.

const nums = [1, 2, 3, 4, 5]; // .map() — transform each element nums.map(n => n * 2); // [2,4,6,8,10] // .filter() — keep elements that pass test nums.filter(n => n % 2 === 0); // [2,4] // .reduce() — fold into single value nums.reduce((acc, n) => acc + n, 0); // 15 // .find() — first element that passes test nums.find(n => n > 3); // 4 // .some() / .every() nums.some(n => n > 4); // true nums.every(n => n > 0); // true // .flat() / .flatMap() [[1,2],[3,4]].flat(); // [1,2,3,4] nums.flatMap(n => [n, n * 2]); // [1,2,2,4,3,6,...] // .sort() — mutates! use with compare fn [3,1,2].sort((a, b) => a - b); // [1,2,3] // Chaining nums .filter(n => n % 2 === 0) .map(n => n ** 2); // [4, 16]
✅ Best Practice Prefer .map(), .filter(), .reduce() over loops for transforming data. They are declarative, composable, and avoid mutation. Always provide the initial value for .reduce().
Array Destructuring & Spread Intermediate
// Destructuring const [first, second, ...rest] = [1, 2, 3, 4]; // first=1, second=2, rest=[3,4] // Swap variables [a, b] = [b, a]; // Spread — copy / merge arrays const combined = [...arr1, ...arr2]; const copy = [...arr]; // Array.from() Array.from("hello"); // ["h","e","l","l","o"] Array.from({ length: 3 }, (_, i) => i); // [0,1,2] // Array.of() Array.of(1, 2, 3); // [1,2,3]

10. Async JavaScript

The Event Loop Advanced

JavaScript is single-threaded. The Event Loop allows it to be non-blocking via asynchronous callbacks.

// Call Stack — synchronous, LIFO execution // Web APIs — browser handles setTimeout, fetch, DOM events // Callback Queue — callbacks from Web APIs wait here // Microtask Queue — Promises go here (higher priority than callbacks) console.log("1"); // sync setTimeout(() => console.log("2"), 0); // macrotask queue Promise.resolve().then(() => console.log("3")); // microtask queue console.log("4"); // sync // Output: 1, 4, 3, 2 // Microtasks (Promises) run before macrotasks (setTimeout)
Callbacks → Promises → Async/Await Intermediate
// 1. Callbacks (old way — leads to "callback hell") getData(id, function(err, data) { if (err) return console.error(err); processData(data, function(err2, result) { ... }); }); // 2. Promises const p = new Promise((resolve, reject) => { setTimeout(() => resolve("done"), 1000); }); p.then(result => console.log(result)) .catch(err => console.error(err)) .finally(() => console.log("cleanup")); // 3. Async/Await (modern, cleanest) async function fetchUser(id) { try { const res = await fetch(`/api/users/${id}`); if (!res.ok) throw new Error("HTTP error"); const data = await res.json(); return data; } catch (err) { console.error("Failed:", err); } }
✅ Best Practice Always use async/await with try/catch. Never let Promises go unhandled — always attach .catch() or wrap in try/catch. Handle errors at the right level.
Promise Combinators Intermediate
// Promise.all — all must resolve, fails fast on any rejection const [user, posts] = await Promise.all([fetchUser(), fetchPosts()]); // Promise.allSettled — waits for all, reports each status const results = await Promise.allSettled([p1, p2, p3]); // Promise.race — first to settle wins const fastest = await Promise.race([fetchFromCDN1(), fetchFromCDN2()]); // Promise.any — first to RESOLVE wins (ignores rejections) const first = await Promise.any([p1, p2, p3]);

17. Design Patterns

Essential JS Patterns Advanced

Module Pattern

Use closures or ES modules to encapsulate private state and expose a public API.

Singleton

Ensure only one instance of a class/object exists. Common in config managers, loggers.

Observer / PubSub

Decouple event emitters from subscribers. addEventListener is observer pattern.

Factory

Create objects without new. Functions that return configured objects.

Strategy

Swap algorithms at runtime. Pass different functions for sorting, validation, rendering.

Proxy Pattern

Intercept object operations with new Proxy(). Used in Vue 3's reactivity.

Decorator

Wrap functions to add behavior. Used in logging, caching, validation.

Command

Encapsulate actions as objects. Enables undo/redo, queuing, logging.

// Observer Pattern Example class EventEmitter { #listeners = new Map(); on(event, fn) { const fns = this.#listeners.get(event) ?? []; this.#listeners.set(event, [...fns, fn]); return () => this.off(event, fn); // unsubscribe } emit(event, ...args) { this.#listeners.get(event)?..forEach(fn => fn(...args)); } off(event, fn) { this.#listeners.set(event, this.#listeners.get(event).filter(f => f !== fn)); } }

19. Performance Optimization

Core Performance Techniques Advanced
  • Debounce — Delay function execution until user stops triggering it (search input, resize).
  • Throttle — Limit function to fire at most once per interval (scroll, mousemove).
  • Lazy loading — Load modules / images only when needed with dynamic import().
  • Memoization — Cache expensive function results for same inputs.
  • Virtual DOM / list virtualization — Only render visible items in long lists.
  • Avoid layout thrashing — Batch DOM reads before writes. Reading offsetWidth then writing style in a loop forces reflow every iteration.
  • requestAnimationFrame — Schedule visual updates to sync with browser repaint cycle.
  • Web Workers — Offload CPU-heavy tasks to background thread.
  • Code splitting — Use bundler (Vite/Webpack) to split code into smaller chunks.
  • Tree shaking — Remove unused exports at build time.
// Debounce implementation function debounce(fn, delay) { let timer; return (...args) => { clearTimeout(timer); timer = setTimeout(() => fn(...args), delay); }; } // Throttle implementation function throttle(fn, interval) { let last = 0; return (...args) => { const now = Date.now(); if (now - last >= interval) { last = now; fn(...args); } }; }

20. JavaScript Security

Common Vulnerabilities & Prevention Advanced

XSS (Cross-Site Scripting)

Injecting malicious scripts via user input. Prevention: Never use innerHTML with user data. Use textContent. Sanitize with DOMPurify.

CSRF

Tricking users into making unwanted requests. Prevention: CSRF tokens, SameSite cookies, verify Origin headers.

Prototype Pollution

Modifying Object.prototype via merge. Prevention: Use Object.create(null) for dictionaries, validate input keys.

Insecure eval()

eval() executes arbitrary strings as code. Prevention: Never use eval(). Use proper parsing/data structures instead.

Sensitive data in JS

API keys in client-side code are exposed. Prevention: Never expose secrets in frontend. Use environment variables on server.

Clickjacking

Embedding your site in an iframe. Prevention: Set X-Frame-Options: DENY or Content-Security-Policy: frame-ancestors 'none'.

✅ Best Practice Apply Content Security Policy (CSP) headers. Subresource Integrity (SRI) for CDN scripts. Validate all input on the server — never trust the client.

Proxy & Reflect Expert

Symbol & Well-Known Symbols Expert
// Custom Symbol const id = Symbol("id"); const user = { [id]: 123, name: "Alice" }; user[id]; // 123 — not enumerable, won't show in for...in // Well-known symbols // Symbol.iterator — makes object iterable // Symbol.toPrimitive — control type conversion // Symbol.hasInstance — customize instanceof // Symbol.toStringTag — customize Object.prototype.toString class MyArray { get [Symbol.toStringTag]() { return "MyArray"; } } Object.prototype.toString.call(new MyArray()); // "[object MyArray]"
Memory Management & Garbage Collection Expert

JS uses automatic garbage collection (mark-and-sweep algorithm). Memory is freed when no references to an object remain. Common leaks:

  • Global variables — accidentally created globals never get collected
  • Detached DOM nodes — DOM removed but JS still holds reference
  • Unremoved event listeners — listeners on removed elements
  • Closures holding large data — inner function keeps outer scope alive
  • Timers not cleared — setInterval without clearInterval
✅ Best Practice Use WeakMap and WeakSet for element-associated data — they don't prevent GC. Always clear intervals/timeouts. Remove event listeners when components unmount (framework lifecycle hooks).
Concurrency Patterns Expert
// Sequential async for (const id of ids) { await processOne(id); // one at a time } // Parallel async (all at once) await Promise.all(ids.map(id => processOne(id))); // Batch (e.g. 5 at a time) async function batchProcess(items, batchSize, fn) { for (let i = 0; i < items.length; i += batchSize) { const batch = items.slice(i, i + batchSize); await Promise.all(batch.map(fn)); } } // Async iteration (streams) for await (const chunk of stream) { process(chunk); }
JavaScript Best Practices — Master List Expert
  • Use const by default, let when needed, never var
  • Always use === strict equality
  • Prefer functional array methods over for loops
  • Write pure functions — avoid side effects where possible
  • Keep functions small and single-purpose (SRP)
  • Use async/await with try/catch for all async code
  • Never mutate function arguments or external state
  • Use named functions instead of anonymous for better stack traces
  • Avoid deep nesting — early returns and guard clauses
  • Document complex logic with comments that explain WHY, not WHAT
  • Set up ESLint + Prettier + TypeScript from project start
  • Write tests before fixing bugs. Aim for 80%+ coverage on business logic
  • Use structuredClone() for deep copies
  • Use AbortController to cancel fetch requests
  • Validate all user input both client AND server side
  • Never log sensitive data (tokens, passwords) to the console
  • Use semantic variable and function names — code is for humans first

25. JavaScript Website Health Check

3 schedules

Systematic maintenance tasks to keep your JavaScript-powered website fast, secure, and reliable. Use this as your operational checklist.

☀️
Daily Checks
Every business day — quick pulse check
Daily
⏱ Time: 15–30 min/day
💪 Effort: Low
💰 Cost: Free (tools already in use)
  • 🔴
    Console Error Monitoring Open browser DevTools Console on key pages. Check for JS runtime errors, uncaught exceptions, and failed network requests. Zero errors should be the daily target.
  • 📡
    Uptime & Availability Verify site loads correctly on desktop and mobile. Check status page if using uptime monitoring (UptimeRobot, Better Uptime — free tier available). Review any downtime alerts from the previous night.
  • ⚡
    Core User Flows Manually test the 2–3 most critical user paths (login, checkout, form submit). Confirm they complete without JS errors or broken UI.
  • 📊
    Error Tracking Dashboard Review Sentry, Rollbar, or Datadog error inbox. Triage any new JS errors that appeared in the last 24 hours. Assign or mark as known.
  • 🔔
    API Health Confirm external APIs your JS depends on (payment gateway, maps, analytics) are returning expected responses. Check their status pages.
  • 📱
    Mobile Smoke Test Quick check on a real mobile device or browser emulator. JavaScript behavior can differ between desktop and mobile browsers (especially Safari on iOS).
📅
Weekly Checks
Every Monday or Friday — deeper review
Weekly
⏱ Time: 1–3 hours/week
💪 Effort: Medium
💰 Cost: Free–$20/mo (Lighthouse CI, monitoring tools)
  • 🚀
    Lighthouse / PageSpeed Audit Run Lighthouse in Chrome DevTools or PageSpeed Insights on your 3 most important pages. Review Performance score — target 90+. Check: First Contentful Paint, Largest Contentful Paint, Total Blocking Time, Cumulative Layout Shift. Identify regressions from the previous week.
  • 📦
    Bundle Size Check Run npm run build and inspect bundle output (Vite's built-in report or rollup-plugin-visualizer). Track total JS bundle size week-over-week. Alert if it grows more than 5% without an intentional new feature.
  • 🔒
    npm Vulnerability Scan Run npm audit. Fix any high or critical severity issues. Run npm outdated to see which packages need updates. Patch minor/patch version updates that are safe. Log in a spreadsheet to track over time.
  • 🧪
    Automated Test Suite Run your full test suite (npm test). Ensure all tests pass. Review test coverage report. If coverage dropped below threshold, add tests before shipping next feature.
  • ♿
    Accessibility Audit Run axe DevTools browser extension or Lighthouse Accessibility audit. Focus on: keyboard navigation, focus management (especially in JS-driven modals/drawers), ARIA attributes, color contrast. Target 90+ score.
  • 📉
    JavaScript Error Rate Trends Review your error monitoring dashboard weekly trends. Compare error count vs sessions. Is your error rate (errors/sessions) stable or increasing? Spike = something broke. Gradual increase = a new code path is buggy.
  • 🌐
    Cross-Browser Testing Use BrowserStack or Sauce Labs to run a quick smoke test across Chrome, Firefox, Safari, and Edge on the latest versions. Focus on JS-heavy interactive components.
  • 🔑
    API Keys & Tokens Review Confirm no API keys were accidentally committed to the repository this week (use git log -p or GitHub's secret scanning). Rotate any that are near expiration.
🗓️
Monthly Checks
First week of each month — strategic review
Monthly
⏱ Time: 4–8 hours/month
💪 Effort: High
💰 Cost: Free–$100/mo depending on tools used
  • 📦
    Full Dependency Upgrade Run npm update and review all changes. Test thoroughly after upgrading. For major version bumps, read changelogs and check for breaking changes. Use npx npm-check-updates to see all available updates at once.
  • 🏗️
    Code Quality Audit Run ESLint over the entire codebase (npx eslint . --ext .js,.ts,.jsx,.tsx). Review and fix all warnings and errors. Consider adding stricter ESLint rules as the codebase matures. Run a code complexity analysis (e.g., SonarQube Community or Code Climate).
  • 🔐
    Security Audit Full npm audit --audit-level=moderate and fix everything. Review your Content Security Policy headers using Security Headers (securityheaders.com). Check for any newly disclosed vulnerabilities affecting your tech stack (CVE databases, framework changelogs). Rotate any long-lived tokens or secrets.
  • 📈
    Performance Benchmarking Run a full Web Vitals report using web-vitals library data from real users (if you have RUM — Real User Monitoring). Compare against previous month. Run a Lighthouse CI audit across all major routes and document results. Check Core Web Vitals in Google Search Console.
  • 🧹
    Dead Code Elimination Identify unused JS files, functions, and npm packages. Use depcheck to find unused dependencies. Remove them. Unused code increases bundle size and maintenance burden.
  • 📝
    Documentation Review Update README, inline JSDoc comments, and API documentation. Ensure setup instructions are still accurate. If using Storybook, review component stories. Documentation debt compounds quickly.
  • 🌍
    Browser Support Matrix Review Check your analytics for actual browser/OS breakdown of your users. Update your .browserslistrc and Babel/PostCSS targets accordingly. Drop support for browsers below a meaningful usage threshold (e.g., < 1%).
  • 🎯
    Tech Debt Sprint Reserve 10–20% of monthly development time for addressing JS tech debt: refactoring complex functions, improving error handling, increasing test coverage, migrating deprecated APIs. Defer no longer than 3 months.
  • 🤖
    CI/CD Pipeline Review Review build times and test durations. Optimize if > 10 min. Check for flaky tests. Review deployment pipeline health. Ensure staging environment reflects production.
  • 🆕
    Ecosystem Updates Read ECMAScript proposal status. Check what's new in the frameworks you use (React, Vue, etc.). Evaluate if any new tool/library should replace something you're currently using. Attend or read summaries from JS community events (JSConf, etc.).
